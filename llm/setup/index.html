配置 AI 开发环境

学这门课之前要先把环境搭好。阿里云 ACP 课程的代码量还挺多的，光看不跑根本理解不深，所以环境这一步不能跳过。

# 这篇讲什么

  • 用什么环境跑课程代码（在线 Notebook vs 本地 IDE）
  • 如何拉取课程代码并安装依赖
  • 常见环境问题怎么解决

# 环境选择

我用阿里云 PAI-DSW 跑的代码，CPU 实例就够，新用户有 3 个月免费额度。

两种在线方案对比：

方案 优点 注意
阿里云 PAI-DSW 新用户 3 个月免费，实例可长时间运行 额度用完后自动按量付费，不用时必须手动停止
ModelScope 免费 Notebook 完全免费，不限次数 单次连续 4 小时后自动关闭，需重启

坑点：PAI-DSW 免费额度耗尽后如果忘记停止实例会自动扣费，我设了个手机提醒，每次用完就去控制台手动停止。

# 获取课程代码并安装依赖

进入 DSW 的 Terminal，先确认 Python 版本和工作目录：

python --version   # 确认是 3.10
pwd                # 确认在 /mnt/workspace
1
2

如果不在 /mnt/workspace，先 cd /mnt/workspace。

# 自动安装（推荐）

wget https://developer-labfileapp.oss-cn-hangzhou.aliyuncs.com/ACP/aliyun_llm_acp_install.sh
/bin/bash aliyun_llm_acp_install.sh
1
2

一键脚本会自动拉代码、创建虚拟环境、安装依赖，成功后直接跳到"切换内核"。

# 手动安装（网络不好时备用）

# 拉代码（网络不好可换 atomgit）
git clone https://github.com/AlibabaCloudDocs/aliyun_acp_learning.git
# 备用：git clone https://atomgit.com/alibabaclouddocs/aliyun_acp_learning.git

# 创建并激活虚拟环境
python3 -m venv llm_learn
source llm_learn/bin/activate

# 安装依赖
pip install --upgrade pip
pip install ipykernel
python -m ipykernel install --user --name llm_learn --display-name "Python (llm_learn)"
pip install -r ./aliyun_acp_learning/requirements.txt

deactivate

# 切换内核

安装完成后，在每个 .ipynb 文件右上角把内核切换到 Python (llm_learn)，否则 import 会报错。

每个课件都要手动切换一次，这步容易忽略。不同章节的依赖版本不一样，用 venv 隔离能避免冲突。

# 常见问题

Q: git clone 超时怎么办？
切换 DSW 实例到其他地域（杭州/张家口）重试，或者用 atomgit 镜像。

Q: 代码仓库有更新，怎么拉最新？

cd aliyun_acp_learning
git checkout .   # 注意：会覆盖本地改动，先备份
git pull
1
2
3

Q: WebIDE 里输入框在哪？
Notebook 模式下输入框在代码块正下方；WebIDE 模式下在代码文件正上方，位置不明显，找不到的话切回 Notebook 更直观。

# 个人总结

  • 环境搭建这步花了我不少时间，主要卡在网络上，建议直接用 atomgit 镜像，比 GitHub 稳定
  • PAI-DSW 的免费额度坑过我一次，用完不手动停就自动扣钱，务必养成用完即停的习惯
  • venv 虚拟环境这个设计很合理，课程各章节依赖版本差异较大，隔离是必要的
  • 每次新打开课件都要切一次内核，这个步骤容易被遗忘，报 ModuleNotFoundError 的时候先检查这个
